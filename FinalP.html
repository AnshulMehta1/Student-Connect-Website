Answering some Commom Questions Related to Prototype 1

Answering some FAQs related to Prototype 1

We used the following Tech Stack

  • Our Development IDE- VSCODE/ Android Studio
  • Framework for building cross Platform Native Application- Flutter
  • Language - Dart
  • User stays Login - Shared Preferences
  • Authentication Methods- Firebase Console
  • Posting Queries in Chatroom- Cloud Firestore
  • For making the Chatroom-Firebase and Cloud Firestore

We improvised on the Second prototype as well as Addes the functionality of Posting Questions un Chatrooms. This process took us about 25 days of Intensive Efforts by the App Leads

  • Structuring the Database
  • Retrieving the List of Questions and queries from Database
  • Adding the Functionality of facilitating multiple amswers to a single question posted in the Groups
  • Bug - User Logs out When App is Closed
  • Bug 2 - Static Searching
  • Bug 3- Unordered Chat Display
  • Bug 4- Static Number of groups (Limited number of groups)
  • Bug 5 - Status Of other users is not being displayed accurately(Online / Offline)
  • Bug 6- Message Status not known (Delivered, Read)
  • Issue - Settings functionality missing

Windows updates haywired the exisitng sdks, our app lead faced a severe after windows updates. We also had to migrate the Project temporarily to another Machine..

Here we write some solutions we implemeted

  • Bug 1 Fix- Used Shared Preferences Package to keep the user logge in
  • Bug 2 Fix- Implemented Elastic Search to ease the searching across the Platform
  • Bug 3 Fix - Ordered The chat stream by time so that the user is provided the integrated view
  • Bug 4 Fix- The Pre-Defined Groups provided in the Product remain same.
